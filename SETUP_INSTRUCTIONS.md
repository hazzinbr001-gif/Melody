# Melody Firebase Setup Instructions

This document provides step-by-step instructions to set up and deploy Melody to Firebase Hosting.

## Quick Start (Automated)

### macOS/Linux

```bash
cd Melody
chmod +x deploy-to-firebase.sh
./deploy-to-firebase.sh
```

### Windows (PowerShell)

```powershell
cd Melody
.\deploy-to-firebase.ps1
```

## Manual Setup

### Step 1: Install Prerequisites

#### macOS (using Homebrew)

```bash
# Install Node.js (includes npm)
brew install node

# Install pnpm
npm install -g pnpm

# Install Firebase CLI
npm install -g firebase-tools
```

#### Windows

1. Download and install Node.js from [nodejs.org](https://nodejs.org/)
2. Open PowerShell or Command Prompt and run:
   ```bash
   npm install -g pnpm
   npm install -g firebase-tools
   ```

#### Linux (Ubuntu/Debian)

```bash
# Install Node.js
sudo apt-get update
sudo apt-get install -y nodejs npm

# Install pnpm
npm install -g pnpm

# Install Firebase CLI
npm install -g firebase-tools
```

### Step 2: Verify Installations

```bash
node --version      # Should show v18.0.0 or higher
npm --version       # Should show 9.0.0 or higher
pnpm --version      # Should show 8.0.0 or higher
firebase --version  # Should show 13.0.0 or higher
```

### Step 3: Clone the Repository

```bash
git clone https://github.com/hazzinbr001-gif/Melody.git
cd Melody
```

### Step 4: Install Dependencies

```bash
pnpm install --ignore-scripts
```

### Step 5: Build for Firebase

```bash
cd artifacts/showcase
npx vite build --config vite.config.firebase.ts
cd ../..
```

### Step 6: Set Up Firebase Project

#### Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click "Add project"
3. Enter project name: `melody-birthday` (or your preferred name)
4. Follow the setup wizard
5. Note your **Project ID** (you'll need this)

#### Initialize Firebase in Your Project

```bash
firebase login
firebase init hosting
```

When prompted:
- Select your Firebase project from the list
- Public directory: `artifacts/showcase/dist`
- Configure as single-page app: `Yes`
- Set up automatic builds: `No`

### Step 7: Deploy to Firebase

```bash
firebase deploy
```

After deployment completes, you'll see:

```
✔ Deploy complete!

Project Console: https://console.firebase.google.com/project/melody-birthday/overview
Hosting URL: https://melody-birthday.web.app
```

Visit the **Hosting URL** to see your live Melody experience!

## Configuration Files

### firebase.json

Located in the project root, this file configures Firebase Hosting:

```json
{
  "hosting": {
    "public": "artifacts/showcase/dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

### .firebaserc

This file stores your Firebase project configuration:

```json
{
  "projects": {
    "default": "melody-birthday"
  }
}
```

Update `melody-birthday` with your actual Firebase project ID.

## Making Changes & Redeploying

To update your live site:

```bash
# 1. Make changes to source code in artifacts/showcase/src/

# 2. Rebuild the project
cd artifacts/showcase
npx vite build --config vite.config.firebase.ts

# 3. Redeploy
cd ../..
firebase deploy
```

## Troubleshooting

### Issue: "pnpm: command not found"

**Solution:**
```bash
npm install -g pnpm
```

### Issue: "firebase: command not found"

**Solution:**
```bash
npm install -g firebase-tools
```

### Issue: "Error: Not logged in"

**Solution:**
```bash
firebase login
```

### Issue: Build fails with module errors

**Solution:** The project has been configured to work with Firebase Hosting. If you encounter module resolution errors:

1. Ensure you're using the Firebase-specific Vite config:
   ```bash
   npx vite build --config vite.config.firebase.ts
   ```

2. Clear node_modules and reinstall:
   ```bash
   rm -rf node_modules
   pnpm install --ignore-scripts
   ```

### Issue: Deployment fails with permission error

**Solution:**
1. Ensure you're logged in: `firebase login`
2. Verify you have access to the Firebase project
3. Check that `firebase.json` exists in the project root

## Advanced Configuration

### Custom Domain

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project → Hosting
3. Click "Connect domain"
4. Follow the domain verification process

### Environment Variables

The Firebase build does **not** use environment variables. If you need environment-specific configuration:

1. Create separate build configurations
2. Use build-time constants in your source code
3. Rebuild and redeploy for each environment

### Performance Monitoring

Enable Firebase Performance Monitoring:

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project → Performance
3. Follow the setup instructions
